# Plan Feature

Use this skill when implementing any meaningful feature.

## Required process
1. Read relevant PRD sections from `/docs/killMyIdea_prd.pdf`
2. Inspect existing files
3. Identify architecture pattern already in use
4. Produce:
   - feature summary
   - affected routes/pages
   - affected components
   - backend/API changes
   - database changes
   - validation rules
   - edge cases
   - risks
   - open questions
5. Stop before coding until approval

## Rules
- Do not jump straight to implementation
- Do not invent hidden requirements
- Keep the plan phased and practical
